후속 조건 반영 · 로컬 문장 다듬기

  • 2차전(후속 칩) — 「아, 진작 말씀하지!」 접두 + 사용자가 고른 조건(예: 5명 이상 단체)을 본문 논점에 녹임 · 로컬 폴백 템플릿도 동일

카카오 후기·평점 데이터 연동 검토(지금은 키워드 검색만), 지도·장소 UX API·정책 확인

추가 기능 히스토리 공유, 알림, 주제 템플릿 등 아이디어 다수 스펙·우선순위 논의 중
기획자로서 — 답변 톤을 논문체에서 구어체로 바꿀 때, “한 줄 프롬프트”가 아니라 금지어·역할·결맺음을 묶어서 고쳐야 체감이 바뀐다는 걸 이번에 몸으로 배움.

바이브코딩 — 가설 → 구현 → 피드백 루프를 짧게 돌리며, 스펙만으로는 안 보이던 분기(맛집/일반/술), 카카오 권한·빈 검색, UI 정렬, 2차전 조건 누락까지 직접 다뤄 봄.
